# HOW MATCHING WORKS:
# Scenarios are checked TOP TO BOTTOM. The FIRST match wins.
# Put your SPECIFIC cases first (e.g., error cases for known IDs).
# Put the DEFAULT catch-all scenario LAST — it catches everything else.
#
# MATCH TYPES:
#   url-contains     — URL path contains a specific string
#   url-regex        — URL path matches a regex pattern
#   body-contains    — Request body (as text) contains a specific string
#   body-json-path   — JSON path in request body equals a value (use: $.field.name == "value")
#   header-equals    — A request header equals a specific value (use: HeaderName == value)
#   always           — Always matches. Use ONLY for SCENARIO DEFAULT at the bottom.
#
