# Banking Session — stateful multi-step flow.
#
# WireMock tracks state per Scenario name. All steps in this file share
# the same scenario name "Banking Session".
#
# State machine:
#   Started ──[Login]──► Authenticated
#   Authenticated ──[Get Account]──► Authenticated
#   Authenticated ──[Initiate Transfer]──► TransferPending
#   TransferPending ──[Confirm Transfer]──► Authenticated
#   Authenticated ──[Logout]──► Started
#
# At runtime:
#   1. WireMock starts with state "Started"
#   2. POST /auth/login   → token returned, state → Authenticated
#   3. GET  /api/v1/account → account data returned, state stays Authenticated
#   4. POST /api/v1/transfers → transfer initiated, state → TransferPending
#   5. PUT  /api/v1/transfers/confirm → confirmed, state → Authenticated
#   6. DELETE /auth/logout → 204, state → Started
#
# Any call to a step whose State-In does not match current state returns 404
# (WireMock finds no matching stub).

Description: Full banking authentication and transfer flow
